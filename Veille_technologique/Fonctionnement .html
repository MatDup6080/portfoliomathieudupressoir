Fonctionnement d'une cyberattaque

Les différentes phases du cycle d'une attaque informatique (Cyber Kill Chain)

1. Reconnaissance

L'attaquant collecte des informations sur sa cible : adresses IP, noms de domaine, employés (LinkedIn, OSINT), technologies utilisées et éventuelles failles connues. C'est une phase passive qui peut durer des semaines sans laisser la moindre trace sur les systèmes visés.

2. Armement

L'attaquant développe ou acquiert un exploit adaptés aux vulnérabilités découvertes : un malware, un ransomware, un kit de phishing ou une backdoor. Il conditionne le payload dans un fichier apparemment anodin (PDF, macro Office, lien URL).

3. Livraison

Le vecteur d'attaque est envoyé vers la cible : e-mail de spear phishing, clé USB abandonnée, compromission d'un site web légitime (watering hole), ou exploitation directe d'un service exposé sur Internet (RDP, VPN vulnérable).

4. Exploitation & Persistance

Une fois le code malveillant exécuté, l'attaquant exploite la faille pour prendre pied sur le système. Il installe ensuite un mécanisme de persistance (clé de registre, tâche planifiée, rootkit) pour survivre aux redémarrages et maintenir l'accès sur le long terme.

5. Mouvement latéral & Exfiltration

L'attaquant se déplace discrètement à travers le réseau (pass-the-hash, kerberoasting), élève ses privilèges jusqu'à l'administrateur de domaine, puis exfiltre les données sensibles vers un serveur C2 (Command & Control) ou chiffre le système pour demander une rançon.
